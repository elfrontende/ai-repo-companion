# Summary

An orchestrator should pick the smallest capable agent set, the lowest safe effort, and the best provider profile for the task.

# Signals

- Reuse stable base agents across chats instead of rebuilding them each time.
- Auto-create specialty agents only when the task repeatedly requires them.
- Route architecture to deep reasoning and routine edits to fast builder profiles.
